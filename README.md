В проектировании моделей баз данных зачастую
применяется формат UML ([Unified Modeling Language](https://en.wikipedia.org/wiki/Unified_Modeling_Language)). Один из
популярных продуктов PlantUML ([website](https://plantuml.com/), [github](https://github.com/plantuml)) предлагает для
этой задачи стандарт [Information Engineering Diagram](https://plantuml.com/ie-diagram).

Чаще всего процесс разработки приложения включает проведение анализа предметной области. Если необходима разработка базы
данных, то выявление набора сущностей и их связей ведет за собой разработку модели, которая далее передается
разработчику (если не была разработана самим разработчиком на предыдущем этапе) в разработку.

Разработчик в свою очередь составляет скрипт (в различных форматах в зависимости от используемого технологического
стека) для создания соответствующих таблиц в базе данных. При разработке логики приложения, под каждую сущность
создаются классы-модели с перечислением всех полей, их свойств и указанием связи этих классов в соответствии с
изначально представленной моделью данных

Цель данного проекта - разработка инструмента(-ов), задача которых автоматизировать некоторые рутинные действия
разработчика, позволяет следующее:

1. Генерировать SQL Data Definition Language (DDL) модель для создания таблиц в бд и проливки изменений под различные
   стандарты от простейших до более
   сложных.
2. Генерировать POJO или заготовок (классов) моделей сущностей также под разные стандарты JDBC или JPA с возможностью
   дальнейшей модификации их и использования в основной логики приложения.

На вход приложения должен поступать файл в одном из расширений PlantUML: .pu, .puml, .txt.

Одним из условий является базовое знание синтаксиса UML. В любом случае, проверить корректность разработки схемы нужно в
PlantUML Web Server (
